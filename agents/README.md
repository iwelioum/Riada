## 🧠 Collective Memory System

The agents share a **persistent memory** that enables continuous learning:

### Memory Structure
```
agents/memory/
├── decisions.md           # Append-only log of all decisions
├── patterns.md            # Extracted reusable patterns
├── tech-watch.md          # Dependency updates & breaking changes
├── error-solutions.md     # Build errors & solutions
└── cycle-reports/         # Reports from each cycle
    ├── cycle-1-audit.md
    ├── cycle-2-backend.md
    └── ... (up to cycle-8)
```

### How Memory Works

1. **During Work:** Agents document decisions in `agents/memory/decisions.md`
   ```
   ## [DATE] [AGENT] — [DECISION TITLE]
   PROBLEM: [issue]
   DECISION: [action taken]
   PATTERN APPRIS: [reusable rule extracted]
   REGRESSION TEST: [test added to prevent regression]
   ```

2. **After 3+ Similar Decisions:** Extract pattern into `agents/memory/patterns.md`
   ```
   ### PATTERN: [Name]
   QUAND: [trigger]
   FAIRE: [steps]
   VÉRIFIER: [validation command]
   EXEMPLE: [real file in project]
   ```

3. **Each Cycle:** EVOLUTION_ENGINE updates agent prompt files with learnings
   - Agents get smarter after every cycle
   - New skills discovered are documented
   - No repeated mistakes (errors catalog prevents regression)

## ❓ FAQ

**Q: How long will this take?**
A: 1-2 weeks intensive (8 cycles, 2-3 days each). Can be compressed or spread out.

**Q: What if an agent finds something blocking another agent's work?**
A: GOVERNOR immediately flags it. Cycle order can be adjusted if needed.

**Q: What if a cycle takes longer than expected?**
A: GOVERNOR reprioritizes. High-impact quick wins are prioritized, tech debt is deferred.

**Q: Can we pause and resume between cycles?**
A: Absolutely! Memory files preserve all decisions. Just read `agents/memory/decisions.md` before resuming.

**Q: What happens after cycle 8?**
A: All knowledge is consolidated. Agent prompts are production-ready. New sessions can directly use upgraded agents without reinvention.
